ğŸ“¦ What Does "IPv4 CIDR Block" Do in VPC?

When you create a **VPC**, AWS asks for an **IPv4 CIDR block**.
This:
- Defines the **total private IP range** available in your VPC
- Subnets inside your VPC will split this range further
**Example:**  
If your VPC is `10.0.0.0/16`, you can create subnets like:

- `10.0.1.0/24`
    
- `10.0.2.0/24`
    
- `10.0.3.0/28`
    

Each of these is a **sub-CIDR** inside the VPC's main CIDR.

### 6. **Network ACL (NACL)**

- Acts like a **firewall at subnet level**
    
- Optional, but powerful
    
- Controls **inbound/outbound rules** for the whole subnet
    
- Unlike Security Groups, NACLs are **stateless** (you must allow both directions manually)
