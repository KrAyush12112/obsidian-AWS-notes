### **The Declarative Way (Docker Compose)**

**Concept:** Instead of typing long commands (Imperative), you write a **configuration file** (Docker file otr YAML) that describes exactly what your application needs. You tell Docker _"This is the result I want,"_ and Docker figures out how to build and run it.

- You **write configuration** instead of typing commands.
- The configuration file defines:
    - Which image to use
    - What ports to expose
    - Volumes to mount
    - Environment variables
    - Restart policies
- Then you run a single command (`docker-compose up` or `kubectl apply`) to create the container(s).
### 3. **Example: Docker Compose (Declarative)**
The language name is - yaml
```
version: '3'
services:
  web:
    image: nginx:latest
    ports:
      - "8080:80"
    volumes:
      - ./html:/usr/share/nginx/html
```

### 4. **Advantages of Declarative Method**

- **Repeatability** â†’ Same file can be used on any machine.
- **Automation** â†’ No need to remember long commands.
- **Scalability** â†’ Easy to define multiple services (web, db, cache).
- **Version Control** â†’ Config files can be stored in Git.

> [!NOTE] There is huge diffrence between YAML And docker file
> Here is the difference:
> 1. **Dockerfile (What your instructor described):** This is the **Recipe**. It tells Docker **how to BUILD** the image. It uses commands like `FROM`, `RUN`, and `CMD`.
> 2. **YAML File (`docker-compose.yml`):** This is the **Manager**. It tells Docker **how to RUN** the container (which ports to open, which volumes to connect).
