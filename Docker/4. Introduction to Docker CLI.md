# 1. All docker command start with DOCKER

# 2. Hirarchical system 
	- Object
		- The docker object that you primarily intract with 
			- 1. image
			- 2. conatiner
			- 3. network
			- 4. volume
	- Command
		- Action you want to perform
			- Build
			- run 
			- pull
			- push
			- logs
			- ls
	- Options
		- Modify command behaviour
			- Usally start with - or --
				- Single Hyphen - 
					- Typically used for short form action.
					- example : docker ps -a {Where -a means -all}
				- Double hyphen --
					- Preffered for lnger name or more descriptive names
					- example : Docker container run --nameOfConatiner
	- Arguments 
		- Specific value of command
			- Image name
			- Image id
	- Examples
		- docker container run ubuntu
		- docker image ls
		- docker volume create --name
