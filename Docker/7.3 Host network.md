## ðŸ§  Definition

- In **host mode**, container apna **network stack host ke saath share karta hai**.
    
- Matlab: container ke paas **apna alag IP nahi hota** (jaise 172.17.x.x bridge me hota hai).
    
- Jo bhi ports container expose karega, wo **directly host ke ports** pe bind ho jaate hain.

| **Port Mapping** | `-p` option kaam nahi karta, kyunki container already host ke ports use karta hai. |     |
| **Isolation**    | Less isolation â€” container aur host ek hi network stack share karte hain.          |     |
## ðŸ”¹ When to Use Host Mode

- Jab tumhe **maximum performance** chahiye (no NAT overhead).
    
- Jab tumhe container ko **host ke network ke saath full integration** dena ho.
    
- Example: monitoring tools (Prometheus node exporter), network sniffers, or services that need raw access to host network.
