## Web Identity Federation

- Web Identity Federation is designed for authenticating users of public-facing applications (web and mobile) with an identity provider they already use

- It leverages(Use) open standards (Protocol) like **OpenID Connect (OIDC)** and **OAuth 2.0**.
### ðŸ” OAuth 2.0
- **Purpose**: Authorization
- **What it does**: Lets apps access your data (like email or photos) without needing your password.
- **Example**: You log into a new app using Google. The app gets permission to see your name and email, but not your password.
### ðŸ†” OpenID Connect (OIDC)
- **Purpose**: Authentication + Authorization
- **What it does**: Confirms who you are (your identity) and can also give access to your data.
- **Example**: You log into a website using Google. OIDC tells the site, â€œThis is really you,â€ and shares your basic profile info.

**How it Works**:
> 	- An end-user signs in to your application using a public IdP like **Google, Facebook, or Login with Amazon**
> 	- The IdP(Google, Facebook) authenticates the user and provides a JSON Web Token (JWT)
> 	- Your application then uses this token (JWT) to call the AWS Security Token Service (STS) and assume a pre-configured IAM role.
> 	- AWS STS validates the token and, if valid, returns temporary security credentials that the application can use to access AWS resources on the user's behalf.

## **Use Cases**:

- This is the ideal solution for consumer-facing mobile and web applications, as it allows users to log in with an identity they already trust.
- You don't need to manage user credentials or passwords, and users can access your application and its AWS resources seamlessly.

# SAML 2.0 Federation - Security assertion markup language
SAML 2.0 Federation is the standard for connecting a company's corporate identity system to AWS.
It's used for **Single Sign-On (SSO)**, allowing employees to access the AWS Management Console or AWS CLI using their existing corporate credentials.

- **SAML** = Standard used for enterprise SSO (Single Sign-On).
- Corporate users log in via their identity provider (like Active Directory Federation Services, Okta, Ping Identity).
- IdP (identity provider) sends a **SAML assertion** to AWS.
